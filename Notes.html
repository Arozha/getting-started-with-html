Important HTML Concepts

How the Web Works

The Internet is a large collection of networks of computers. When you go to www.google.com in your browser, your computer first sends out a request to find the IP address of the name you typed in. This is called a DNS (Domain Name System) request. Every computer has a default DNS server they can send the request to. After receiving the reply from the DNS server, your computer finally is ready to send an HTTP request.

HTTP (HyperText Transfer Protocol) is the protocol used to display web pages in browsers. Your computer sends the request to its default gateway (typically your home router). Then the gateway forwards the request to your internet provider and after a number of hops through different routers, the HTTP request makes it to the web server, which sends data back through to your computer. That sure sounds like a lot of traveling to do! Luckily, with fiber-optic cabling and ultra-fast servers, this whole process only takes a few seconds.

HTML

HTML stands for HyperText Markup Language. Every time you visit a web site, you're seeing HTML as it is interpreted by your browser. HTML documents contain two types of content: text content - which is what you see - and markup - which determines how the text content is organized and styled.

Tags and Elements

HTML elements are the building blocks of HTML documents. Browsers differentiate between elements by using tags. Tags surround text to create elements. Most elements need opening and closing tags, however, some, called void tags, don't have any text at all.

Inline vs Block Elements

Inline elements are those HTML elements that are "on the line". They have no additional spacing. <span> and <br> are examples of inline elements. Block elements have a height and width of space around their content. <div> and <p> are examples of block elements.

Computers are Stupid

Computers only understand clear, unambiguous instructions. They also interpret them literally and cannot infer things that we humans can. For example, if you were to tell a computer how to drive, everything would have to be explained. Left, right, forwards, and backwards would be meaningless to a computer unless clearly defined by a programmer. Things that are simple to us are actually very difficult when we think about the algorithms necessary to relate them to machines.
